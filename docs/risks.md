# Risks

## 1. 内部 API の安定性
- **リスク**: StreetVoice の API は公式公開されておらず、予告なく変更・廃止される可能性がある
- **影響**: 大 - 全機能が動作しなくなる可能性
- **対策**: repository パターンで API 呼び出しを隔離済み。API 変更時は datasource 層のみ修正すればよい
- **備考**: API v4/v5/v6 が共存しており、段階的に移行されている兆候あり

## 2. Rate Limiting
- **リスク**: 明示的な rate limit ヘッダは観測されていないが、過度なリクエストでブロックされる可能性
- **影響**: 中 - 一時的な利用不可
- **対策**: リクエスト頻度を抑える。キャッシュ層の追加を検討

## 3. HLS URL の寿命
- **リスク**: HLS URL に署名パラメータは含まれていないが、サーバー側で失効制御される可能性
- **影響**: 中 - 再生中断
- **対策**: 再生前に毎回 URL を取得する設計にしている。長期間の URL キャッシュは避ける
- **検証結果**: 現時点では URL 失効は観測されていない

## 4. 利用規約
- **リスク**: StreetVoice の利用規約に内部 API の利用制限がある可能性
- **影響**: 大 - プロジェクト自体の継続不可
- **対策**: 個人利用 PoC に限定。公開配布はしない

## 5. 地理的制限
- **リスク**: 一部コンテンツや API が地域制限されている可能性
- **影響**: 小〜中 - 特定曲の再生不可
- **対策**: エラーハンドリングで対応

## 6. HLS の MP3 コーデック互換性
- **リスク**: HLS ストリームが MP3 ベース (mp4a.40.34) であり、一部デバイスでデコード問題が起きる可能性
- **影響**: 小 - Media3 ExoPlayer は MP3 HLS を標準サポート
- **対策**: ExoPlayer が対応しているため問題なし

## 7. APK サイズ
- **リスク**: Debug APK が 70MB と大きい
- **影響**: 小 - PoC なので問題なし
- **対策**: Release ビルドで minify 有効化、不要な依存の整理

## 継続価値の判断

### 結論: **継続価値あり**

### 根拠
1. **全フロー成功**: 検索 → 曲詳細 → HLS URL 取得 → 再生可能な M3U8 取得まで、未ログインで全て成功
2. **認証不要**: Cookie, CSRF, ブラウザセッションいずれも不要
3. **標準的な HLS**: Media3 ExoPlayer で直接再生可能な標準形式
4. **豊富な API**: 検索以外にも、ランキング、プレイリスト、ジャンルなど拡張の余地が大きい
5. **APK ビルド成功**: Android TV アプリとして実機デプロイ可能な状態

1. Google TV 実機で APK をインストール・動作確認
2. 実際に 1 曲再生して音声出力を確認
3. DPAD ナビゲーションの動作確認
4. ランキング/プレイリスト閲覧機能の追加
5. 連続再生（プレイリスト再生）の実装
6. エラーハンドリングの改善
7. Release ビルドの最適化
